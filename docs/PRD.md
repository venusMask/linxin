# 灵信 (LinXin) - AI原生即时通讯应用 PRD (MVP)

## 1. 产品定位
**灵信**是一款以AI为核心驱动力的即时通讯工具。除了基础的聊天功能外，它通过开放的 **Agentic API** 允许第三方智能体（如 OpenClaw, AutoGPT 等）以语义化的方式自动执行操作。

### 核心差异点
- **语义化操作**：第三方 Agent 无需知道具体的 UserID，通过“给媳妇发信息”等自然语言指令即可触发业务逻辑。
- **安全隔离**：建立三层架构，在保护用户隐私（不暴露完整通讯录）的同时提供强大的自动化能力。
- **多端同步**：Agent 从云端发出的指令，移动端实时回写并同步展示。

---

## 2. 核心用户场景

### 场景 A：移动端本地 AI 辅助 (Local Agent)
用户在移动端直接通过语音或文字下令：“帮我问问老李明天几点开会。”
- **流程**：App 调用服务端 AI 解析意图 -> App 本地执行 `sendMessage` -> UI 实时更新。

### 场景 B：第三方 Agent 远程调用 (Remote Agent)
用户授权 OpenClaw 访问灵信，并在 OpenClaw 界面输入：“帮我告诉爱人，我今晚加班。”
- **流程**：OpenClaw 携带 Token 调用灵信云端接口 -> 服务端进行语义解析（“爱人” -> ID: 1001）-> 服务端执行发送 -> 服务端通过 WebSocket 推送给用户的所有移动设备 -> 移动端回写消息记录。

### 场景 C：语义冲突处理 (Ambiguity Handling)
用户指令：“给张三发信息。” 但通讯录中有两个张三（张三-同事，张三-同学）。
- **流程**：系统解析发现冲突 -> 返回 `REQUIRE_CLARIFICATION` 状态及脱敏后的提示 -> Agent 询问用户。

## 3. 功能列表 (MVP)

### 3.1 基础通讯
- [x] **即时消息**：支持文字消息发送、接收、历史记录查看。
- [x] **好友管理**：添加/删除好友，支持备注名修改（语义解析的关键数据源）。
- [x] **群组功能**：基础群聊创建与消息发送。

### 3.2 AI 与 Agent 能力 (核心)
- [ ] **能力发现接口 (Manifest API)**：暴露系统支持的自动化工具描述（JSON Schema）。
- [ ] **语义解析层 (NLU Layer)**：
    - 私有化实体匹配：将自然语言中的称呼/备注映射为系统内部 ID。
    - 冲突检测：识别歧义并请求澄清。
- [ ] **自动化执行接口**：支持携带 Token 的远程调用。

### 3.3 数据同步
- [ ] **WebSocket 实时回写**：确保云端操作能实时推送至移动端并入库。
- [ ] **增量消息同步**：客户端上线后自动拉取离线期间的所有变动。

---

## 4. 安全与隐私要求
- **数据最小化暴露**：禁止第三方 Agent 直接通过接口拉取用户完整的通讯录。
- **Token 作用域 (Scopes)**：Agent 必须经过用户授权后才能获得特定权限（如 `msg:send`）。
- **溯源标记**：Agent 发出的所有消息在 UI 上必须有明确的来源标记。

---

## 5. 验收标准
1. 第三方 Agent 能够通过 Token 调用接口，成功给指定备注的好友发送消息。
2. 移动端 App 在上述操作完成后，无需手动刷新即可在聊天界面看到该消息。
3. 如果指令中的联系人不存在或有歧义，接口能返回对应的错误/提示码。
