# 前端功能文档

## 核心功能模块

### 1. 认证功能

#### 1.1 用户登录
- **页面**: login_page.dart
- **功能**: 用户登录获取Token
- **输入**:
  - 用户名
  - 密码
- **流程**:
  1. 输入用户名和密码
  2. 点击登录按钮
  3. 调用AuthService.login()
  4. 保存Token到SharedPreferences
  5. 跳转到主页
- **错误处理**:
  - 用户名或密码错误
  - 网络异常
  - 服务器错误

#### 1.2 用户注册
- **页面**: register_page.dart
- **功能**: 注册新用户账号
- **输入**:
  - 用户名
  - 密码
  - 确认密码
  - 昵称
  - 手机号（可选）
  - 邮箱（可选）
- **流程**:
  1. 填写注册信息
  2. 表单验证
  3. 调用AuthService.register()
  4. 注册成功后跳转登录页
- **验证规则**:
  - 用户名：3-20字符，字母数字下划线
  - 密码：6-20字符
  - 昵称：2-20字符

#### 1.3 自动登录
- **功能**: 应用启动时自动登录
- **流程**:
  1. 检查SharedPreferences中的Token
  2. Token有效则自动登录
  3. Token无效则跳转登录页

#### 1.4 退出登录
- **功能**: 用户退出登录
- **流程**:
  1. 清除SharedPreferences中的Token
  2. 清除用户信息
  3. 断开WebSocket连接
  4. 跳转登录页

## 版本历史

### v1.0.1 (2026-03-31)

#### 🐛 BUG修复

##### 1. MessageService单例模式问题（中等）
**问题描述**:
- 使用可空类型和空合并运算符实现单例
- 可能导致多次实例化
- Dart单例模式不规范

**修复方案**:
- 使用标准的Dart单例模式
- 添加`factory`构造函数
- 使用`static final`确保唯一实例

**影响文件**: `message_service.dart`

**代码示例**:
```dart
class MessageService {
  MessageService._();

  static final MessageService instance = MessageService._();

  factory MessageService() => instance;

  // ... 其他代码
}
```

**修复前**:
```dart
class MessageService {
  static MessageService? _instance;
  static MessageService get instance => _instance ??= MessageService._();

  MessageService._();
}
```

**修复后**:
```dart
class MessageService {
  MessageService._();

  static final MessageService instance = MessageService._();

  factory MessageService() => instance;
}
```

---

##### 2. WebSocket重连延迟计算错误（中等）
**问题描述**:
- 使用`pow(2, _reconnectAttempts).toInt()`计算延迟
- 增长过快，第5次就达到32秒
- 应该使用更合理的指数退避算法

**修复方案**:
- 使用位运算`1 << _reconnectAttempts`实现指数退避
- 更合理的重连延迟（1s, 2s, 4s, 8s, 16s, 30s）

**影响文件**: `websocket_service.dart`

**代码示例**:
```dart
void _scheduleReconnect() {
  if (!_shouldReconnect) return;

  _reconnectTimer?.cancel();
  final delay = Duration(seconds: min(_maxReconnectDelay, 1 << _reconnectAttempts));
  _reconnectAttempts++;

  LogService.info('WebSocket reconnecting in ${delay.inSeconds}s (attempt $_reconnectAttempts)');

  _reconnectTimer = Timer(delay, () async {
    await _doConnect();
  });
}
```

**重连延迟对比**:

| 重连次数 | 修复前 | 修复后 |
|---------|--------|--------|
| 1 | 2s | 1s |
| 2 | 4s | 2s |
| 3 | 8s | 4s |
| 4 | 16s | 8s |
| 5 | 32s | 16s |
| 6 | 64s | 30s |

---

#### 📝 改进说明

1. **单例模式规范化**: MessageService使用标准Dart单例模式，避免多次实例化
2. **重连策略优化**: WebSocket重连延迟更合理，避免过长的等待时间

---

#### 🔧 技术细节

##### Dart单例模式最佳实践
- **私有构造函数**: `MessageService._()`
- **静态实例**: `static final MessageService instance = MessageService._()`
- **工厂构造函数**: `factory MessageService() => instance`
- **优点**: 
  - 线程安全
  - 延迟初始化
  - 简洁明了

##### 指数退避算法
- **公式**: `delay = min(maxDelay, 2^attempt)`
- **实现**: `1 << _reconnectAttempts` (位运算)
- **优点**:
  - 避免频繁重连
  - 逐步增加延迟
  - 有最大延迟限制

### v1.0.2 (2026-03-31)

#### 🐛 BUG修复

##### 1. WebSocketService单例模式不规范（轻微）
**问题描述**:
- WebSocketService使用旧的可空单例模式
- 与MessageService的单例模式不一致
- 代码风格不统一

**修复方案**:
- 改为标准Dart单例模式，与MessageService保持一致
- 使用`static final`确保唯一实例
- 移除dispose()中不需要的单例重置代码

**影响文件**: `websocket_service.dart`

**代码示例**:
```dart
class WebSocketService {
  WebSocketService._();

  static final WebSocketService instance = WebSocketService._();

  factory WebSocketService() => instance;

  // ... 其他代码
}
```

**修复前**:
```dart
class WebSocketService {
  static WebSocketService? _instance;
  
  factory WebSocketService() {
    _instance ??= WebSocketService._internal();
    return _instance!;
  }

  WebSocketService._internal();

  void dispose() {
    // ...
    _instance = null;  // 不需要重置
  }
}
```

**修复后**:
```dart
class WebSocketService {
  WebSocketService._();

  static final WebSocketService instance = WebSocketService._();

  factory WebSocketService() => instance;

  void dispose() {
    // ...
    // 移除了不需要的单例重置代码
  }
}
```

---

#### 📝 改进说明

1. **代码一致性**: WebSocketService与MessageService使用相同的单例模式
2. **代码简洁**: 移除了不必要的单例重置代码
